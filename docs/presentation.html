Design Highlights

  • Full 3NF normalization — no redundancy
  • Self-referencing categories for hierarchy
  • FK constraints enforced on all foreign keys
  • Inventory tied to both product AND warehouse
  • Payments & Shipping separate from Orders
  • Returns linked to order_items (line-level)
  • Auto-total via AFTER trigger on order_items

Sample Advanced Query (Inactive Sellers Trend)
SELECT TO_CHAR(ms.month, 'YYYY-MM'),
  COUNT(DISTINCT s.seller_id) AS inactive_count
FROM generate_series(...) AS ms(month)
CROSS JOIN sellers s
WHERE s.seller_id NOT IN (
  SELECT DISTINCT p.seller_id
  FROM products p
  JOIN order_items oi ON p.product_id = oi.product_id
  JOIN orders o ON oi.order_id = o.order_id
  WHERE o.order_date BETWEEN p_start_date
    AND (ms.month + '1 month - 1 day')::DATE
)
GROUP BY ms.month
ORDER BY ms.month;
